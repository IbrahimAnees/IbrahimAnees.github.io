SM3 Job Solver

Algorithmic solution to a job-scheduling problem.

Tech stack: Java, Maven, JavaFX, GraphViz, Figma

Repo: https://github.com/ibrahimanees/sm3-solver/

The problem

The input is a directed acyclic task graph G = (V, E, w, c), where V is the set of tasks and E is the set of edges between the tasks. A weight w associated with task n represents its execution time and weight c represents the communication time if two tasks are executed on different processors. If they are executed on the same processor, the communication time is 0.

The objective of the problem is to minimise the schedule length which is the finish time of the last task.

Algorithm

During development, both the A* and branch and bound algorithms were implemented to find an optimal solution to the problem. However for the final version, A* is used as it is proven to be faster. Pruning techniques were used to speed up the algorithm.

Parallelisation

In the A* algorithm, we go through each partial schedule within the priority queue of schedules. Due to this, we can split tasks in the priority queue as evenly as possible for each thread so that they can run the rest of the algorithm on the remaining partial schedules. This is how SM3 implements parallelisation with different cores.

Visualisation

The SM3 visualiser is designed in Figma, and developed using JavaFX. The node graph is shown using the GraphViz library.
